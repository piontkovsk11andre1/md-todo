# 168. Reach Tools and Health from main menu

## Problem

The main menu defined in 158 has six items: `Continue, New Work, Workers, Profiles, Settings, Help`. The Tools (164) and Health (165) scenes exist as standalone scenes but have no entry path from this menu. Without a defined navigation graph, those scenes are unreachable in the integrated TUI.

Adding two more top-level menu items would dilute the retro-game feel and push the menu past the comfortable "six options" cognitive load. The scenes belong as drilldowns from the most natural parent.

## Goal

Define and implement the navigation graph that makes Tools and Health reachable without expanding the main menu.

## Model

Drilldown wiring:

- **Workers (162) → Health (165)**: register `[H]` (shift-h) hotkey on the Workers scene. The Workers scene already shows the per-candidate health column; `[H]` opens the full Health scene as a child. `Esc` returns to Workers.
- **Workers (162) → Tools (164)**: register `[T]` (shift-t) hotkey on the Workers scene for the `commands.tools.<toolName>` overrides relationship. `Esc` returns to Workers.
  - Rationale: the per-tool command override (`commands.tools.<name>`) is a worker-routing concern. Tools scene is the natural deeper view from there.
  - Alternative considered: drilldown from Settings. Rejected because Settings is a generic config viewer and Tools needs prompt-editing affordances that don't belong in Settings.

Lowercase `h`/`t` remain available for scene-local actions (none today, but keeping single-key namespace clean).

The footer hotkey hints on the Workers scene gain two entries:

```
[e] edit config   [H] health   [T] tools   [r] reload
```

The main menu's right-column status text for `Workers` now also mentions reachability:

```
Workers   default ✓ · 2 fallbacks · 1 cooling   (H: health · T: tools)
```

This is the discoverability hook for the drilldown.

## Implementation boundaries

Files expected to change:

- `implementation/src/presentation/tui/scenes/workers.js` — add `[H]` and `[T]` keybindings.
- `implementation/src/presentation/tui/scenes/main-menu.js` — extend the Workers row right-column hint.
- `implementation/src/presentation/tui/navigation.js` (introduced in 158 as the scene router) — formalize a stack of scenes so child scenes can `Esc` back to the parent rather than always returning to the main menu.

Out of scope:

- Adding Tools / Health as top-level main-menu rows.
- Breadcrumb UI in scene headers.
- Deep-link CLI flags like `rundown --scene health`.

## Acceptance

- From the main menu, selecting Workers and pressing `H` opens Health; `Esc` returns to Workers, not to the main menu.
- The same applies to `T` opening Tools.
- The Workers scene footer and the main menu Workers status row both surface the `H`/`T` hints.
- The scene stack correctly handles arbitrary depth: main → Workers → Health → Esc → Workers → Esc → main.
