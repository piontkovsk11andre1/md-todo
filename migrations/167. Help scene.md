# 167. Help scene

## Problem

The TUI has no in-app way to read the docs, see keybindings, or jump to project resources. Users have to leave the TUI, open the repo, and grep `docs/`. That breaks the "workshop you walk into" feel and pushes new users out of the app at the moment they need help most.

## Goal

Build a Help scene that lists local docs, synthesized references, external links, and a keybindings cheatsheet. Local files open in a built-in pager (read-only). External URLs open in the system browser. Editing is intentionally not offered — Help is reading, not authoring.

## Model

Scene layout:

```
┌─ Help ──────────────────────────────────────────────────────────────┐
│                                                                     │
│  Documentation (local)                                              │
│    [↵] README.md                                                    │
│    [↵] roadmap.md                                                   │
│    [↵] docs/configuration.md                                        │
│    [↵] docs/cli-config.md                                           │
│    [↵] docs/cli-worker-health.md                                    │
│    [↵] docs/cli-<command>.md  (one row per discovered file)         │
│                                                                     │
│  Synthesized references                                             │
│    [↵] AGENTS.md template (live, via getAgentsTemplate())           │
│    [↵] Effective config dump  (current workspace)                   │
│                                                                     │
│  External                                                           │
│    [o] Project website                                              │
│    [o] Changelog                                                    │
│    [o] Issue tracker                                                │
│                                                                     │
│  [k] Keybindings cheatsheet                                         │
│                                                                     │
│  [↵] view in pager   [o] open in browser   [Esc] back               │
└─────────────────────────────────────────────────────────────────────┘
```

Behavior:

- Local doc rows are discovered by globbing `docs/*.md` from the workspace root, plus the fixed `README.md` / `roadmap.md` entries. Missing files are hidden, not shown as broken rows.
- `[↵]` opens the entry in `pager.js` (created in 164). The pager supports up/down line scroll, page up/down, `g`/`G` for top/bottom, `/` placeholder for future search, and `q` / `Esc` to close.
- `AGENTS.md template` row renders the output of `getAgentsTemplate()` from [src/domain/agents-template.ts](implementation/src/domain/agents-template.ts) directly into the pager.
- `Effective config dump` renders the same content the Settings scene shows under `effective` scope, but as a single concatenated text block.
- `[o]` on external rows shells out to the system "open URL" command:
  - macOS: `open <url>`
  - Linux: `xdg-open <url>`
  - Windows: `start "" <url>`
  - Fallback: print the URL in a banner with "open this URL manually".
- `[k]` opens an overlay listing the global keybindings (Esc back, ↑↓ navigate, : palette in 170 if present, etc.) and any scene-specific keys when the user came from a scene that registered them.

External URL list is hand-maintained in the scene file; do not embed URLs anywhere outside this module to keep the source of truth obvious.

## Implementation boundaries

Files expected to change:

- new `implementation/src/presentation/tui/scenes/help.js`.
- new `implementation/src/presentation/tui/components/keybindings-overlay.js`.
- reuse `pager.js` from 164.
- reuse `getAgentsTemplate()` from existing domain code.

Out of scope:

- Pager search.
- Bookmarks / history of recently-viewed docs.
- Editing local docs from the TUI.

## Acceptance

- Opening Help on a workspace that lacks `roadmap.md` does not show that row.
- `[↵]` on a local doc opens the file in the pager and renders its content.
- `[↵]` on `AGENTS.md template` opens the synthesized template in the pager.
- `[o]` on an external row launches the OS browser; the TUI remains alive and re-renders on return.
- `[k]` shows the keybindings overlay; `Esc` closes it.
- The scene works on a freshly cloned workspace with no `.rundown/` directory.
