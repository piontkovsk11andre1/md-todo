# 155. Move cli-sketch into presentation tui scaffold

## Problem

`implementation/src/cli-sketch.js` is a working TUI prototype that imports from `../dist/index.js` (the bundled output) because it lives outside the TS source tree. Its current location signals "throwaway sketch", but we are committing to integrate it as the real `rundown` interactive front end.

Before any behavior changes, the file needs a permanent home that is consistent with the rest of the presentation layer and that lets it import from typed source rather than the build output.

## Goal

Relocate the sketch into `implementation/src/presentation/tui/` as plain JavaScript (a `.js` ESM file), wired through the existing `presentation` import paths. Keep behavior identical. No CLI surface change in this migration.

## Model

After this migration:

- `implementation/src/cli-sketch.js` is deleted.
- `implementation/src/presentation/tui/index.js` exists with the same code, with imports adjusted:
  - `import { createApp } from "../../dist/index.js"` becomes `import { createApp } from "../../create-app.js"` (or whichever in-tree module exposes `createApp`).
  - All other relative paths fix up to the new location.
- A new bin entry or npm script keeps an easy way to run the sketch standalone for local iteration:
  - `implementation/package.json` `scripts.tui-sketch` invokes `node ./src/presentation/tui/index.js`.
- The file remains a Commander-driven entry with the same flags it has today (`--fps`, etc.).
- No tests, no Commander wiring into the main `cli.ts` yet. That is intentional and lands in 157.

## Implementation boundaries

Files expected to change:

- delete `implementation/src/cli-sketch.js`.
- add `implementation/src/presentation/tui/index.js` (move + import-path adjustments).
- update `implementation/package.json` scripts.
- update `implementation/.gitignore` or `tsup.config.ts` only if they reference the old path.
- update `implementation/README.md` if it mentions `cli-sketch.js`.

Out of scope:

- TypeScript port (lands in 168).
- Any change to behavior, menu items, or output handling.
- Wiring into the published `rundown` binary.

## Acceptance

- `node ./implementation/src/presentation/tui/index.js` boots the sketch identically to before.
- `npm run tui-sketch` from `implementation/` does the same.
- `grep -R "cli-sketch" implementation/` returns no hits.
- `grep -R "../dist/index.js" implementation/src/presentation/tui/` returns no hits.
