# 177. Remove AGENTS Starter File and Agents Output Mode

## Problem

The current product surface still treats `AGENTS.md` as a first-class starter artifact and exposes a dedicated CLI output mode for it.

Today that shows up in three connected places:

- `rundown start` scaffolds `AGENTS.md` into the starter directory.
- Root and agent help flows expose `--agents` to print canonical `AGENTS.md` guidance.
- CLI docs, help text, tests, and TUI help still describe `AGENTS.md` as an expected user-facing workflow.

That behavior is now the wrong default. `AGENTS.md` is a weird fit for the starter scaffold, and keeping a dedicated command just to print template guidance adds product surface area without enough value.

## Desired behavior

After this migration:

- `rundown start` does not create `AGENTS.md`.
- `--agents` is removed from the root command.
- `--agents` is removed from `rundown agent`.
- User-facing docs, help text, error messages, TUI help, and tests stop referring to `AGENTS.md` as a supported feature.

The product should no longer present `AGENTS.md` generation or management as a built-in rundown workflow.

## Behavior contract

### 1. Start no longer scaffolds `AGENTS.md`

Project initialization should create the rundown workspace structure only.

- Do not write `AGENTS.md` during Start.
- Do not reserve special starter-dir behavior for AGENTS guidance.
- Re-running Start must not recreate `AGENTS.md`.

This is a starter-surface cleanup only. It does not change other scaffolded directories or files.

### 2. Remove `--agents` from the CLI surface

The dedicated AGENTS output mode should no longer exist.

- Remove root `rundown --agents`.
- Remove `rundown agent --agents`.
- Remove any parsing, routing, or precedence logic that special-cases `--agents`.
- Remove command descriptions that mention printing canonical AGENTS guidance.

Once removed, the CLI help surface should not advertise AGENTS-specific output anywhere.

### 3. Remove AGENTS-specific product messaging

Documentation and in-product help should stop treating AGENTS guidance as a named feature.

- Remove AGENTS-specific command docs such as the dedicated `cli-agents` documentation.
- Remove references in general CLI docs and README navigation.
- Remove TUI help entries that open or preview an AGENTS template.
- Remove wording in errors or guidance that instructs users to call `rundown --agents` or `rundown agent --agents`.

If agent-related guidance still needs to exist anywhere, it should be expressed through normal docs, not a scaffolded file or a special output command.

## Scope

In scope:

- Stop Start from scaffolding `AGENTS.md`.
- Remove the CLI `--agents` flag from both root and `agent` paths.
- Remove AGENTS template output helpers and any dedicated template plumbing that only serves this feature.
- Remove or rewrite docs/help/test coverage that mention AGENTS.md generation.
- Update integration and presentation tests to reflect the removed feature.

Out of scope:

- Replacing `AGENTS.md` with another starter file.
- Adding a different export or print command for agent guidance.
- Broad changes to the rest of the interactive help or TUI flows beyond removing AGENTS-specific behavior.
