Remove bracketed/boxed gutter chrome from human CLI logs so grouped run output stays readable when copied into markdown, notes, or follow-up tasks.

## Problem statement

Grouped CLI output currently renders with box-drawing gutter characters such as `┌`, `│`, and `└`. That terminal chrome makes copied logs noisy and reads like bracketed indentation rather than useful status text. The grouping itself is still useful; the decorative gutter is not.

## Scope

### In scope

- Flatten grouped human CLI output in `implementation/src/presentation/output-port.ts` so grouped lines use plain leading spaces instead of box-drawing gutter characters.
- Preserve grouped readability by keeping indentation depth for nested groups and grouped multiline/stdout/stderr output.
- Preserve message content, timestamps, counters, and success/failure icons.
- Update presentation tests that assert the grouped CLI output shape.

### Out of scope

- Changing JSONL/global log event payloads.
- Changing task label formatting such as `[n/m]`, `[#k]`, or timestamp brackets.
- Redesigning TUI rendering or non-CLI presentation surfaces.

## Acceptance criteria

- Grouped CLI output no longer prints `┌`, `│`, or `└` in terminal logs.
- Nested groups remain readable via plain-space indentation.
- Grouped multiline `text` and `stderr` output remains aligned with the active group depth.
- Existing message text, timestamps, counters, and pass/fail semantics remain unchanged.
- Presentation tests cover the flattened output shape.
