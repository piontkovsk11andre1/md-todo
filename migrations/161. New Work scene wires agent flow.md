# 161. New Work scene wires agent flow

## Problem

The `New Work` row in the main menu (158) opens the existing agent flow as-is. That flow assumes `.rundown/agent.md` exists and that a worker is configured. On a fresh checkout neither is true, and the user is dropped into a confusing failure. The row's status probe (159) already warns about it; the scene must complete the loop and offer recovery.

## Goal

Make `New Work` self-healing for first-time users while preserving the fast path for configured workspaces.

## Model

On scene entry, run the same checks as the New Work probe and branch:

- **`agent.md` missing** → show a panel:

  ```
  No agent prompt found.

  rundown looks for the agent prompt at .rundown/agent.md.
  This file controls how rundown talks to the AI in agent sessions.

   [g]  Generate from template (writes .rundown/agent.md)
   [o]  Open .rundown/ in your editor
   [Esc] Back to menu
  ```

  `g` writes the canonical AGENTS.md template (`getAgentsTemplate()` from [src/domain/agents-template.ts](implementation/src/domain/agents-template.ts) — already exists) and re-runs the checks.

- **No worker configured** (`workers.default` and `workers.tui` missing, no CLI worker) → show a panel pointing the user at `rundown init --worker <command>` and offering to launch it interactively. A first version can simply print the suggested command and an `[Esc]` hint.

- **No worker eligible** (config present but worker-health says cooling/unavailable across all candidates) → show the failing entries and a `[r]eset health` action that calls the relevant API.

- **Ready** → enter the agent run view, identical to today's "Open Agent" / "Discuss the Agent" flows. Two sub-actions on entry:
  - `Enter` opens agent (uses `.rundown/agent.md`),
  - `d` opens discuss-agent (uses `.rundown/discuss-agent.md`).

## Implementation boundaries

Files expected to change:

- `implementation/src/presentation/tui/scenes/new-work.js` — refactor.
- `implementation/src/presentation/tui/components/missing-agent-panel.js` — new.
- Reuse existing `getAgentsTemplate()` and existing agent invocation paths.

Out of scope:

- Building an interactive `rundown init` wizard inside the TUI — emit the command for the user to run instead. A later migration may absorb it.

## Acceptance

- On a workspace with no `.rundown/agent.md`, opening New Work shows the recovery panel.
- Pressing `g` writes the template, the panel disappears, and the agent run view becomes available.
- On a fully configured workspace, opening New Work goes straight to the agent run view in one keystroke.
- Pressing `d` opens the discuss-agent variant.
